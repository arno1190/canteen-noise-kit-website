# Messagerie GTM — pilote portail (CAN-60 / impl. CAN-63)

Aligné sur la **garde CAN-58** : aucune promesse de date, de périmètre produit ou de fonctionnalités portail non arbitrées par le board. Le site public décrit un **pilote sur invitation**, pas un portail général disponible.

## Narratif bi-piste (paragraphe + puces)

**Aujourd’hui**, les établissements et collectivités s’appuient sur des **supports hors ligne et tangibles** : guides PDF, firmware, kits et ateliers. C’est la **source de vérité opérationnelle** pour déployer le dispositif en cantine, sans dépendre d’un compte en ligne.

**En parallèle**, un **pilote portail** peut être proposé **sur invitation** à certains partenaires B2G : il s’agit d’une **phase d’exploration** (contenus, accompagnement, retours utilisateurs), **distincte** du catalogue PDF / kit. Le pilote **n’est pas** un remplacement immédiat des PDF ; les deux pistes coexistent tant que le board n’a pas figé la feuille de route publique.

- **Piste A — Maintenant :** PDF, kits, communauté, devis / commande via les canaux indiqués sur le site.
- **Piste B — Pilote (invitation) :** espace en ligne **éventuel**, **sans calendrier annoncé** ici ; intérêt à exprimer via **contact / devis** pour évaluation avec la direction.

## Flags juridique / confidentialité (pour relecture board)

- Préciser que les **données personnelles** traitées dans un futur pilote feront l’objet d’**informations dédiées** (PAI / contrat) avant inscription.
- Ne pas afficher de **sous-traitants** ou **pays d’hébergement** tant que la fiche n’est pas validée juridiquement ; renvoyer vers **équipe / devis** pour les collectivités.
